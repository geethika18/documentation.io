Kaizala Shared Code

React Native and C++ Shared Code repository for Kaizala.

Setup

1. Node

1.1. Windows
  1. Install Chocolatey.
  2. Install Node using Chocolatey. Open a Command Prompt as Administrator, then run:
    choco install nodejs.install
choco install python2
1.2. Mac
  1. Install Homebrew.
  2. Install Node and Watchman using Homebrew. In a Terminal, run:
    brew install node
brew install watchman

2. React Native and TypeScript

Install the React Native CLI and TypeScript. In a Terminal run:

npm install -g react-native-cli
npm install -g typescript

3. Android/iOS repositories

3.1. Prerequisite - Generate Git Credentials
  1. Go to the MOIApps_Android repository.
  2. Click on the 'Clone' button (right-top)
  3. Click Generate Git Credentials:

You will need the Credentials to set up the repository. If you lose it, you can generate one again.

3.2. Windows
  1. Install Git. Open a Command Prompt as Administrator, then run: 
    \\modropserv\Tools\MOSetUp\MOSetUp.bat
    The script will install Git (if it is not already) which will require a restart. After the restart run the command again in a Command Prompt as Administrator to resume the script.
  2. When prompted, enter the full path to a working directory where the MOIApps_Android repository will be cloned.
  3. When prompted, enter the Git Credentials generated above.
3.3. Mac

Clone the MOIApps_Android repository. In a Terminal run:

git clone https://office.visualstudio.com/DefaultCollection/OC/_git/MOIApps_Android

Clone the MOI_iOS repository. In a Terminal run:

git clone https://office.visualstudio.com/DefaultCollection/OC/_git/MOI_iOS

4. MOI submodule

4.1. Android (Windows and Mac)
  1. Clone the MOI submodule. In a Terminal run:
    cd MOIApps_Android
git submodule init
git submodule update --remote
  2. Setup the submodule. In a Terminal run:
    cd KaizalaS/kaizalaS
npm install
4.2. iOS (Mac only)
  1. Clone the MOI submodule. In a Terminal run:
    cd MOI_iOS
git submodule init
git submodule update --remote
  2. Setup the submodule. In a Terminal run:
    cd Apps/kaizalaS/kaizalaS
npm install

5. Android Studio

5.1. Windows

The Windows enlistment setup will install Android Studio for you.

  1. Find and run the mo_enlist Desktop shortcut as Administrator.
  2. Wait for the Nuget to download missing packages (first time).
  3. Type androidstudio to launch the IDE.
  4. Choose Custom setup instead of Standard and install the missing components mentioned below.
5.2. Mac
  1. Install Android Studio from the Android developer website.
  2. Create/edit the .profile in your Home folder. Add the following alias:
    polymer()
{
 ~/<path-to-working-directory-from-Home>/MOIApps_Android/tools/bin/set_env.py
}
  3. Restart Terminal and run polymer to launch Android Studio
  4. After running Android Studio for the first time choose Custom setup and install the missing components mentioned below.

If you get the 'command not found' error when running polymer, then Terminal might be reading the .bash_profile instead of .profile. Move the alias defined in step 2 to .bash_profile and restart Terminal.

5.3. Additional requirements

Download the latest versions of the following components from the Android SDK manager:

  • Android SDK Platform 25 (Nougat)
  • Android SDK Build-Tools
  • CMake
  • LLDB
  • Android SDK Platform Tools
  • Android SDK Tools
  • NDK
  • Android Support Repository
  • Google Repository

6. Xcode (Mac only)

Install Xcode from the App Store.

7. Visual Studio Code (optional)

If you plan on working on React Native, install Visual Studio Code. It has complete TypeScript integration, which we use for React Native development.

8. SourceTree (optional)

SourceTree is an alternative to using the Git CLI. Though it is not as powerful, most Git operations can be performed using SourceTree.

Build

Android
  1. Launch Android Studio using the methods mentioned earlier.
  2. After the Gradle sync, check that you can see the C++ files under the kaizalaS module:

    If you can't then select Build > Refresh Linked C++ Projects from the Toolbar.
  3. Build the project. First time build will take a long time (~30 mins).
  4. Deploy Kaizala:

Pro Tip: Improve build time:

  1. Find the number of cores on your machine.
  2. Navigate to your Android SDK folder. The location can be found from Android Studio Settings > Appearance & Behaviour > System Settings > Android SDK.
  3. Go to the ndk-bundle folder.
  4. Edit ndk-build and replace the line @echo off %~dp0\build\ndk-build.cmd % with @echo off %~dp0\build\ndk-build.cmd -jN %* where N is the number of cores. For example, @echo off %~dp0\build\ndk-build.cmd -j4 %* if you have 4 cores.
